# Fiscal Plugin

`@run-iq/plugin-fiscal` provides universal tax calculation models applicable to any tax system worldwide. It ships with 6 calculation models, 3 meta-models, jurisdiction resolution, and fiscal breakdown reporting.

## Installation

```bash
npm install @run-iq/plugin-fiscal
```

Peer dependencies: `@run-iq/core`, `@run-iq/plugin-sdk`.

## Overview

| Property | Value |
|---|---|
| **Name** | `@run-iq/plugin-fiscal` |
| **Version** | `0.1.0` |

| **License** | Source-Available |

## Models

| Model | Formula | Returns |
|---|---|---|
| `FLAT_RATE` | `base * rate` | `number` |
| `PROGRESSIVE_BRACKET` | Cumulative bracket calculation | `CalculationOutput` |
| `MINIMUM_TAX` | `MAX(base * rate, minimum)` | `CalculationOutput` |
| `THRESHOLD_BASED` | Tax above/at threshold | `CalculationOutput` |
| `FIXED_AMOUNT` | Constant amount | `number` |
| `FIXED_AMOUNT` | Returns a static amount, commonly used for stamp duties or fixed fees. |
| `COMPOSITE` | Aggregates multiple sub-models using `SUM`, `MIN`, `MAX`. |

See [Calculation Models](/docs/concepts/models) for detailed documentation of each model.

## beforeEvaluate Hook

The fiscal plugin's `beforeEvaluate` hook performs five operations:

1.  **Meta-rule condition evaluation**: Evaluates DSL conditions for meta-rules against the raw input data. 

<Callout type="info" title="Full DSL Power">
  Meta-rule conditions are no longer parsed manually. They use the **full DSL engine** (e.g., JsonLogic, CEL) registered in the engine context. This allows for complex logic within inhibitions and substitutions.
</Callout>

2.  **Meta-rule processing**: Passes the rules and condition results through the `<Tag>MetaRuleProcessor</Tag>`, which short-circuits, inhibits, or substitutes regular fiscal rules.
3.  **Short-circuit handling**: If `META_SHORT_CIRCUIT` matched, immediately returns an empty rules array and a special marker.
4.  **Jurisdiction precedence**: Resolves priorities for remaining rules based on jurisdiction level and scope.
5.  **Country filtering**: Pre-filters rules matching the context's country code.

**Base scores:**

| Jurisdiction | Base Score |
|---|---|
| `NATIONAL` | 3000 |
| `REGIONAL` | 2000 |
| `MUNICIPAL` | 1000 |

**Scope multipliers:**

| Scope | Multiplier |
|---|---|
| `GLOBAL` | 1.0 |
| `ORGANIZATION` | 1.1 |
| `USER` | 1.2 |

**Priority = `round(base * multiplier)`**

Full priority table:

| | GLOBAL | ORGANIZATION | USER |
|---|---|---|---|
| **NATIONAL** | 3000 | 3300 | 3600 |
| **REGIONAL** | 2000 | 2200 | 2400 |
| **MUNICIPAL** | 1000 | 1100 | 1200 |

Higher priority rules are evaluated first. A NATIONAL+USER rule (3600) overrides a NATIONAL+GLOBAL rule (3000).

If `input.meta.context.country` is provided, only rules matching that country code are kept:

```typescript
const country = input.meta.context?.country; // e.g., 'TG'
const filtered = rules.filter(r => r.country === country);
```

## Meta Models

Meta-models don't perform calculations themselves — they manipulate the execution pipeline during the `beforeEvaluate` phase, altering the set of rules that are eventually evaluated. They are processed sequentially by a deterministic state-free `<Tag>MetaRuleProcessor</Tag>`.

| Model Name | Description |
|---|---|
| `META_SHORT_CIRCUIT` | Halts execution immediately. If the rule's condition is met, returns an empty rules list and a specified value. |
| `META_INHIBITION` | Removes specific rules from the execution pipeline based on IDs, categories, or tags. |
| `META_SUBSTITUTION` | Dynamically overrides the parameters of existing rules (e.g., swapping out a rate for targeted rules). |

Meta-models are processed during `beforeEvaluate` to modify the returned rules array. Rules removed via `META_INHIBITION` are not passed to the core engine, guaranteeing they are skipped.

## afterEvaluate Hook

The fiscal plugin enriches the result with a `fiscalBreakdown` — contributions grouped by `category`:

```typescript
// result.meta.fiscalBreakdown
{
  TVA: 900000,
  IRPP: 625000,
  IS: 500000,
}
```

This makes it easy for the host application to display tax breakdowns by category.

## FiscalRule Extended Fields

`FiscalRule` extends the base `Rule` with fiscal-specific fields:

```typescript
interface FiscalRule extends Rule {
  jurisdiction: FiscalJurisdiction;  // 'NATIONAL' | 'REGIONAL' | 'MUNICIPAL'
  scope: FiscalScope;               // 'GLOBAL' | 'ORGANIZATION' | 'USER'
  country: string;                  // ISO 3166-1 alpha-2 (e.g., 'TG', 'FR')
  category: string;                 // Tax category (e.g., 'TVA', 'IRPP', 'IS')
}
```

<Callout type="warning" title="Meta-rules are pre-evaluated">
  Meta-rules are processed exclusively in the fiscal plugin's `beforeEvaluate` hook — modifying the rule array returned as `BeforeEvaluateResult`. The core engine has no knowledge of meta-rule semantics, it simply executes the rules the plugin hands back to it.
</Callout>

## Real-World Examples (CGI Togo 2025)

### IRPP — Progressive Brackets

```typescript
{
  id: 'tg-irpp-2025',
  model: 'PROGRESSIVE_BRACKET',
  params: {
    base: 'income',
    brackets: [
      { from: 0,         to: 500_000,   rate: 0 },
      { from: 500_000,   to: 1_000_000, rate: 0.10 },
      { from: 1_000_000, to: 3_000_000, rate: 0.15 },
      { from: 3_000_000, to: 5_000_000, rate: 0.25 },
      { from: 5_000_000, to: null,      rate: 0.35 },
    ],
  },
  jurisdiction: 'NATIONAL',
  scope: 'GLOBAL',
  country: 'TG',
  category: 'IRPP',
  effectiveFrom: new Date('2025-01-01'),
  effectiveUntil: null,
  tags: ['irpp', 'income-tax'],
}
```

## PluginDescriptor

The fiscal plugin exports a `fiscalDescriptor` providing metadata for tooling:

```typescript
import { fiscalDescriptor } from '@run-iq/plugin-fiscal';

fiscalDescriptor.name;            // '@run-iq/plugin-fiscal'
fiscalDescriptor.domainLabel;     // 'fiscal'
fiscalDescriptor.ruleExtensions;  // jurisdiction, scope, country, category
fiscalDescriptor.inputFields;     // revenue, income, expenses, netProfit
fiscalDescriptor.examples;        // VAT, IRPP, minimum tax examples
fiscalDescriptor.promptGuidelines; // model selection guidance
```

The descriptor powers:
- MCP server tool schemas
- CLI rule validation
- Playground input forms
- LLM prompt context for rule generation
