# Introduction

Run-IQ is a **parametric rule engine** for regulated domains — tax, payroll, compliance, insurance. It evaluates declarative rules against structured input and produces deterministic, auditable results.

## Why Run-IQ?

Regulated domains demand guarantees that generic rule engines don't provide:

- **Determinism** — same rules + same input = same output, always
- **Immutability** — rules and input are never mutated during evaluation
- **Auditability** — every evaluation produces a trace and can be persisted as an integrity-verified snapshot
- **Decimal precision** — all monetary calculations use `decimal.js`, never floating-point
- **Reproducibility** — snapshots capture the full evaluation context (rules, DSL versions, params) for exact replay

## Architecture at a Glance

```
Host Application
  └── PPEEngine
        ├── Plugins        (domain logic: fiscal, payroll, ...)
        ├── DSL Evaluators (JSONLogic, CEL, ...)
        ├── Pipeline       (10-step evaluation)
        └── Snapshot       (persistence + integrity)
```

The **engine core** is domain-agnostic. It knows nothing about tax brackets or payroll calculations. All domain logic lives in **plugins** that register calculation models. Rule conditions are evaluated by pluggable **DSL evaluators**.
