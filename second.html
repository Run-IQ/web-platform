The core has zero knowledge of any domain. Rules are versioned data stored by the Host. Plugins register calculation models at startup.

The engine knows no domain.

rule.model is an opaque string. rule.params is unknown. The core resolves, dispatches, and traces — nothing more.

Every calculation leaves a sealed record.

Full rule data — not IDs — are copied into the snapshot. A SHA-256 checksum seals it. Any calculation can be replayed exactly 5 years later.

Conditions are data, not code.

Rules carry optional DSL conditions — JSONLogic or CEL — stored as JSONB in your database. No deployment required to activate or deactivate a rule.

Idempotence is architectural.

Every call carries a requestId. If already processed, the existing snapshot is returned immediately. Network retries and BullMQ re-queues are safe.

4 — Guarantees

What the engine commits to.

Determinism

Same input. Same result. Always.

A computation from 2021 is mathematically identical when replayed today. Same inputs, same engine version, same result — guaranteed by construction.

Immutability

Rules cannot mutate in transit.

Rules are strictly read-only throughout the entire pipeline. A SHA-256 checksum is verified at every reception — any tampering is detected immediately.

Idempotence

A requestId runs once. Period.

Network retries and worker re-queues never produce double calculations. The engine checks for an existing snapshot before executing anything.

Legal Reproducibility

Replay any calculation, years later.

Snapshots store full rule objects — not IDs. Engine version, plugin versions, DSL evaluators. Everything needed to reproduce the exact result is permanently archived.

Decimal Precision

No floating-point surprises.

Arithmetic precision is enforced in every financial calculation. Native float representations are strictly prohibited to ensure totals are perfectly predictable.

Snapshot Integrity

No snapshot. No result.

In strict mode, a snapshot save failure throws before the result is returned. A calculation without an archived legal trace simply does not exist.

5 — Plugin Ecosystem

Domain knowledge,

The engine is MIT and free. Domain plugins carry the regulated knowledge — fiscal brackets, payroll rules, social contributions. They are Source-Available: readable, auditable, commercially licensed.

Checksum strategy

Host computes SHA-256 once on write. Core compares strings only — zero CPU during pipeline.

Phase 3 optimization

AOT compilation of DSL conditions into native JS functions. Target: 10–50× faster condition evaluation.
